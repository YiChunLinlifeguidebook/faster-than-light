# 超越光速：來解決複雜的工作流程
> Status: WIP / Public Timestamp Edition
> Project: Mechanical Soul / 機械靈魂
> Purpose: Public verification record without exposing black-box logic

## 一、公開目的

本文件為 Mechanical Soul 專案之「公開存證版」。
目的不是公開全部核心，而是先保留：

- 研發時間戳
- 概念演進證據
- 架構起點
- 與後續會議室 / 多 AI 協作系統的連動關係
- 可驗證但不洩露黑盒的公開版本

本文件發布後，後續仍會持續補強與迭代。

---

## 二、目前公開可驗證的部分

### 1. 決策引擎骨架
目前已公開的架構骨架包含：

- Data Sources
- Ingestion Layer
- Normalization Layer
- Feature Extraction Layer
- Indicator Engine
- Scoring Layer
- Decision Interface
- Output / API / Logs

### 2. 系統定位
此核心不是一般聊天工具或一般資料看板，而是：

- 用來處理複雜流程
- 加速決策
- 供後續多 AI 協作系統接入
- 能與任務分派、提案收斂、紀錄層、共享記憶層對接的底層核心

### 3. 與會議室系統的連動
後續將接入：
- 多 AI 協作會議室
- 任務分工區
- Proposal / Final Decision
- Recorder / 紀錄層
- 自動化工作流
- 共享記憶層

---

## 三、可驗證依據

本公開存證版可透過以下內容交叉驗證：

1. GitHub 倉庫提交時間
2. 本文件 commit 紀錄
3. `docs/architecture.md` 架構文件
4. `docs/sketches/` 內的 UI 草圖
5. 會議室與決策引擎相關討論的持續演進脈絡
6. 後續版本之差異比對

此版本的目的，是建立一個可追溯、可對照、可持續補強的起點。

---

## 四、刻意不公開的黑盒部分

以下內容目前不在本公開版中揭露：

- 具體權重分配
- 具體評分公式
- 私有 prompt / handoff prompt
- 任務守門閾值
- 去重規則細節
- 冷卻秒數策略
- Ghost Auditor 觸發條件
- Heartbeat / retry / reassign 內部策略
- n8n / workflow 實際調度邏輯
- 內部 JSON 協議細節
- 個人化映射規則與黑盒對應表

也就是說：

**公開的是「架構存在、方向存在、起點存在、演進存在」；**
**保留的是「真正讓系統跑出差異的內核」。**

---

## 五、公開原則

本公開版遵守以下原則：

1. 可驗證
   - 能對照時間、檔案、提交紀錄、後續版本

2. 不洩密
   - 不公開真正核心算法與黑盒規則

3. 可延續
   - 後續版本可接續補強，不與本版衝突

4. 可連動
   - 可作為會議室系統、自動化工作流、共享記憶層的前置起點
